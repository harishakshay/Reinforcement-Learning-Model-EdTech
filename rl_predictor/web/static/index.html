How the Simulation Works

State -> Action -> Reward -> Policy update, tuned for intentional discovery and reduced algorithm fatigue.

State Signals (10-D): Exact RL Input Vector

  • [0] Novelty Score
  • [1] Repeat Exposure
  • [2] Interest Momentum
  • [3] Creator Diversity
  • [4] Session Fatigue
  • [5] Topic Drift
  • [6] Engagement Depth
  • [7] Serendipity Potential
  • [8] Bubble Risk
  • [9] Cognitive Load
  • Input format: 10-feature float vector (used every step by the RL policy)

Policy Loop + Action IDs

  • Action 0 -> Diversify Feed
  • Action 1 -> Stay Balanced
  • Action 2 -> Deepen Interest
  • Exploration level shifts action preference using key signals: state[0], state[6], state[8]
  • Baseline comparator score: 0.4 * state[1] + 0.4 * state[2] + 0.2 * state[0]

Reward Philosophy: Exact Numeric Rules

  • If prediction == 1 and actual == 1 -> reward = +0.5
  • If prediction matches actual direction -> +1.0
  • Early intervention bonus (is_early) -> +0.5
  • High confidence bonus (confidence > 0.8) -> +0.5
  • Opposite-direction miss -> -1.0
  • Near miss (balanced vs directional) -> -0.2

Live Metrics Exposed During Simulation

  • prediction, prediction_label, confidence
  • actual_trend, actual_label
  • reward, total reward, accuracy, streak
  • reward_breakdown.decision_match (0/1)
  • reward_breakdown.early_diversity_bonus (0/1)
  • reward_breakdown.high_confidence_bonus (0/1)
  • reward_breakdown.mismatch_penalty (0/1)
  • reward_breakdown.total (numeric)
